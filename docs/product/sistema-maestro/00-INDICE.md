\# Sistema Maestro V2 — Índice documental



\## Estado



Capa documental canónica para definir la evolución del producto antes de seguir implementando frontend, backend, billing, créditos, Builder, exportación, deploy e IA.



\## Objetivo



Cerrar la doctrina de producto para que la siguiente fase técnica se haga sin improvisación, sin contradicciones, sin deuda estructural y sin prometer capacidades que el sistema todavía no pueda sostener.



Este índice ordena la capa de producto de Sistema Maestro V2 y fija la lectura correcta de sus documentos activos.



\---

Sistema Maestro no debe avanzar como suma de pantallas, informes o respuestas aisladas.



Debe avanzar como sistema completo:



```text

entrada
